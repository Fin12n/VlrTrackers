Tên sản phẩm
VALORANT Tracker

  Thể loại
Website thông tin / Tracker Tool
Single-Page Application (SPA) — Không cần tải lại trang
Ý nghĩa tên gọi
VALORANT — Tên game FPS 5v5 nổi tiếng của Riot Games. Đây là đối tượng dữ liệu mà website tập trung vào.
Tracker — Có nghĩa là "công cụ theo dõi". Website giúp người dùng theo dõi số liệu, thứ hạng và lịch sử thi đấu.
Kết hợp lại: VALORANT Tracker = Công cụ theo dõi dữ liệu game VALORANT.
Phần 02
Ý tưởng & nguồn gốc
Ý tưởng ban đầu
Xuất phát từ nhu cầu thực tế
Em là người chơi VALORANT và thường xuyên muốn xem stats của bản thân hoặc bạn bè mà không cần phải khởi động game (game rất nặng, mất 3–5 phút).
"Em muốn làm một website có thể tra cứu rank, KDA và lịch sử đấu chỉ bằng cách nhập Riot ID — không cần đăng nhập, không cần mở game."
Nguồn cảm hứng
Tracker.gg — Website tracker phổ biến nhất cho nhiều tựa game. Em thường xuyên dùng nó và nảy ra ý định: "Tại sao mình không tự làm?"
tracker.gg yêu cầu đăng nhập Riot Account và có nhiều quảng cáo.
UI trong game VALORANT — Giao diện trong game rất đẹp và đặc trưng với tông màu đỏ đen, font chữ Rajdhani. Em muốn tái hiện phong cách đó lên web.
HenrikDev API — Em tình cờ tìm thấy một API miễn phí cung cấp dữ liệu Valorant → đây là yếu tố quyết định để bắt tay vào làm.
Phần 02 — tiếp theo
Tại sao chọn ý tưởng này?
Đam mê & gần gũi
Em yêu thích VALORANT nên có nhiều hiểu biết về game, dữ liệu, và biết người chơi cần gì. Điều đó giúp em thiết kế sát thực tế hơn.
Học được kỹ năng mới
Đây là lần đầu em làm dự án có kết nối API bên ngoài. Em muốn thử sức với Fetch API, JSON parsing và xử lý dữ liệu bất đồng bộ (async/await).
Sản phẩm có thể dùng được thật
Khác với bài tập "giả định", sản phẩm này chạy với dữ liệu thực tế và em có thể chia sẻ cho bạn bè dùng ngay sau khi hoàn thành.
Phạm vi vừa sức
API có sẵn, không phải tự thu thập dữ liệu → giảm thiểu rủi ro kỹ thuật, phù hợp với thời gian học.
Giao diện thú vị
Design theo phong cách game cyberpunk/tactical cho em nhiều cơ hội sáng tạo CSS nâng cao.

Icon system + Rajdhani, Barlow Condensed, Share Tech Mono typography.

Phần 04 — tiếp theo
Khó khăn gặp phải
Thách thức kỹ thuật
API v3 → v4 thay đổi cấu trúc
HenrikDev nâng cấp API giữa chừng làm toàn bộ code xử lý JSON bị sai. data.players đổi từ flat array sang nested, teams từ object sang array, stats.damage từ số thành object {dealt, received}.
CORS & API rate limit
Gọi API quá nhiều lần bị lỗi 429 (Too Many Requests). Phải tối ưu lại số lần gọi, tránh gọi trùng lặp không cần thiết.
Async/await và race condition
Một số phần render UI chạy trước khi fetch API xong → dữ liệu hiển thị rỗng. Phải kiểm soát thứ tự thực thi cẩn thận hơn.
Thách thức thiết kế
clip-path phức tạp
Các góc cắt xiên (góc cắt Valorant) yêu cầu tính toán clip-path chính xác, rất khó debug khi responsive.
Responsive trên nhiều màn hình
Grid layout phức tạp bị vỡ trên mobile. Phải viết lại nhiều media query và ẩn bớt cột thông tin.
Bài học rút ra: Luôn đọc kỹ documentation trước khi code, và kiểm tra API response thực tế thay vì tin vào tài liệu cũ.

Phần 07 — Bắt buộc
Hạn chế & tính năng tương lai
Hạn chế hiện tại
API rate limit (429 Too Many Requests)
Free tier của HenrikDev giới hạn request/phút. Tra cứu nhiều liên tiếp sẽ bị chặn tạm thời.
Chỉ xem được 7 trận gần nhất
API free tier giới hạn size response. Không thể load thêm trận cũ hơn hay phân tích xu hướng dài hạn.
Bảo mật auth còn yếu
Dùng btoa() (Base64) — không mã hóa thực sự. Phù hợp học tập, không dùng cho production.
Không có caching
Mỗi lần reload phải gọi API lại từ đầu, tốn request và chậm hơn cần thiết.
Kế hoạch phát triển
Hoàn thành — Match Detail & Scoreboard
Bảng điểm đầy đủ, stats chi tiết, hero banner map.
Sắp tới — Pagination match history
Load thêm trận đấu cũ (page 2, 3...) thay vì chỉ 7 trận đầu.
Sắp tới — localStorage caching
Cache kết quả API trong 5 phút để giảm request và tăng tốc độ.
Tương lai — Radiant Leaderboard
Bảng xếp hạng top Radiant mỗi khu vực, cập nhật theo mùa.
Tương lai — Backend thực sự
Node.js + PostgreSQL, bcrypt auth, so sánh 2 người chơi, lưu lịch sử tra cứu yêu thích.
